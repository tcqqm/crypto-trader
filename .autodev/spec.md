# 加密货币自适应交易系统

## 目标指标
- 夏普比率 > 1.5
- 最大回撤 < 15%
- 胜率 > 45%
- 盈亏比 > 1:2
- 月均交易 30-60 笔

## 交易标的
BTC/USDT, ETH/USDT, SOL/USDT, BNB/USDT, XRP/USDT

## 技术框架
Freqtrade + FreqAI + ccxt + Claude API

## 核心策略：市场状态自适应

### 市场状态识别
- ADX > 25 → 趋势市场 → 趋势跟踪策略
- ADX < 20 → 震荡市场 → 均值回归策略
- ATR top 20% → 高波动 → 降低仓位50% + 宽止损
- BB 收窄 → 低波动 → 等待突破

### 趋势跟踪子策略
- EMA 交叉（快线 9，慢线 21）
- 多时间框架确认（5m + 1h）
- 移动止盈（ATR 倍数）

### 均值回归子策略
- RSI 超买超卖（30/70）
- 布林带边界反弹
- 固定止盈

### LLM 情绪层
- 每30分钟抓取加密新闻
- Claude 评分 -1 到 +1
- 情绪 < -0.5 → 不开新多仓
- 情绪 > 0.5 + 看多 → 加大仓位

## 风控规则（硬编码）
- 单笔止损: -2%
- 单笔止盈: +4%（移动止盈）
- 盈亏比: 最低 1:2
- 单笔仓位: 总资金 10-20%
- 同时持仓: 最多 3 个
- 日最大亏损: -5% → 当日停止
- 连续亏损: 5笔 → 暂停24h
- 情绪否决: 市场情绪 < -0.5 → 不开多仓

## 迭代规则
1. 每轮必须运行完整回测（6个月数据）
2. 记录所有指标到 results/backtest_history.json
3. 与上一轮对比，只保留改进的版本
4. 避免过拟合：必须在 train/test 分割上验证
5. 达到目标指标后进入模拟交易阶段
6. 每 5 轮迭代执行一次 Edge 发现流程（见下方）

## Edge 策略自动发现管线（每 5 轮迭代触发）

### 流程（7 阶段）
1. **观察**：分析最近回测结果，提取异常模式（高胜率时段/币种/市场状态）
2. **线索提取**：识别潜在 edge（如：ETH 在亚洲时段趋势信号胜率显著高于平均）
3. **概念合成**：将线索转化为可测试的策略假设
4. **策略设计**：编码新的子策略逻辑（作为 AdaptiveStrategy 的新分支）
5. **审查**：检查是否存在未来函数、数据泄露、过拟合风险
6. **回测验证**：独立回测新子策略，对比基准
7. **信号聚合**：有效策略加入自适应切换池，无效策略丢弃

### 触发条件
- 每 5 轮常规迭代后自动触发
- 或当连续 3 轮迭代指标无改善时提前触发

### Edge 发现数据源
- `results/backtest_history.json` — 历史回测指标
- `results/trade_memory.db` — 交易记忆（按市场状态/币种/时段分类的胜率）
- `results/validation_report.json` — Walk-forward 验证结果

### 验收标准
- 新子策略必须在 walk-forward 验证中表现稳定（train/test 差异 < 30%）
- 新子策略加入后整体夏普比率不下降
- 参数稳定性测试通过（±20% 变动，收益变化 < 30%）

### 子策略管理
- 有效子策略记录到 `results/edge_strategies.json`
- 每个子策略包含：名称、触发条件、适用市场状态、历史胜率
- 自适应切换逻辑根据当前市场状态选择最优子策略组合

## 质量标准
- 无未来函数/数据泄露
- 过高收益需警惕过拟合
- 参数微调不应导致大幅变化
- 覆盖牛/熊/震荡市场环境
